Tips & Best Practices — Stitch + Antigravity

หัวข้อBest Practice
Stitch Promptระบุ platform, color scheme, font, และ accessibility requirement ให้ครบ
Design Consistencyสร้าง DESIGN.md ก่อนเสมอ — Agent ทุกตัวจะยึด file นี้
Antigravity Instructionsใช้ 'senior-level' instructions เช่น 'ใช้ TypeScript strict mode' ได้ code สะอาดกว่า
Planning Modeเปิด Planning Mode ก่อน Agent จะแสดง plan รอ approve — ลดการทำงานผิด
MCP Stitchเรียก @stitch.generate_screen สำหรับ screen ใหม่ โดยไม่ต้องกลับไปเปิด browser
Iterationแก้ทีละส่วน อย่าส่ง prompt เปลี่ยนทุกอย่างพร้อมกัน — Agent จะ confused
Git Checkpointgit commit ก่อนสั่ง mission ใหญ่ทุกครั้ง
Test Firstใส่ 'รัน tests หลังแก้ไขทุกครั้ง' ใน prompt — Antigravity จะทำให้อัตโนมัติ
🆕 Dynamic Subagentsบอกให้ Agent ใช้ subagents สำหรับงานที่ทำพร้อมกันได้ เช่น "สร้างทั้ง 3 หน้าพร้อมกัน ใช้ subagents คู่ขนาน"
🆕 Scheduled Tasksตั้ง background task สำหรับงานซ้ำ เช่น "รัน QC report ทุกวันเที่ยงคืน" — ไม่ต้องเปิด app
🆕 Voice Commandsกดไมค์แล้วพูด prompt ภาษาอังกฤษได้เลย — เหมาะกับ brainstorm ขณะเดิน
🆕 CLI Modeantigravity agent run "task" รัน Agent ใน terminal ได้ — เหมาะกับ CI/CD pipeline
