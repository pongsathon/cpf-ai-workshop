Claude Code — AI Coding Agent

เขียนโค้ด แก้บัค และสร้าง Automation Script ด้วย AI ใน Terminal โดยไม่ต้องเป็น Programmer มืออาชีพ

⏱ 60 นาที Node.js Required Terminal / CLI 🆕 New Module
🤖 Claude Code คืออะไร?

Claude Code คือ AI Coding Agent จาก Anthropic ที่ทำงานใน Terminal โดยตรง ต่างจาก Chatbot ทั่วไปตรงที่มันสามารถ อ่านไฟล์ในโปรเจกต์จริง → แก้ไขโค้ด → รันคำสั่ง → Commit Git ได้ทั้งหมดด้วยตัวเอง ผ่านภาษาธรรมชาติ ไม่ต้อง copy-paste

📂 อ่าน Codebase
เข้าใจโปรเจกต์ทั้งหมด ไม่ใช่แค่ไฟล์เดียว
✏️ แก้ไขไฟล์จริง
เขียน บันทึก ลบ ไฟล์ได้โดยตรง
🔁 รัน & ทดสอบ
รัน script, test และ Git ได้เลย
💡 เหมาะกับ CPF เพราะ — ทีม IT, Data, QC, Logistics สามารถใช้ Claude Code เขียน Python Script วิเคราะห์ข้อมูลโรงงาน, อ่าน Excel, สร้าง Report อัตโนมัติ โดยไม่ต้องเป็น Developer เต็มตัว

ขั้นตอนที่ 1 — ติดตั้ง Claude Code

1
ตรวจสอบ Node.js (ต้องเวอร์ชัน 18 ขึ้นไป)
Claude Code ต้องการ Node.js v18+ เป็น runtime
Terminal
$ node --version v20.11.0 # ✅ OK — ต้องเป็น v18+ $ npm --version 10.2.4 # ✅ OK
⚠️ ถ้าไม่มี Node.js ให้ดาวน์โหลดจาก nodejs.org เลือก LTS version แล้วติดตั้งก่อน
2
ติดตั้ง Claude Code ด้วย npm
ติดตั้งครั้งเดียว ใช้ได้ทุก project
Terminal
$ npm install -g @anthropic-ai/claude-code added 45 packages in 12s $ claude --version Claude Code 2.x.x
💡 ถ้าเห็น permission error บน macOS ให้ใช้ sudo npm install -g @anthropic-ai/claude-code
3
เปิดใช้งานและ Login
รัน claude ครั้งแรก แล้ว login ด้วย Anthropic account
Terminal
$ claude ╭──────────────────────────────────────────╮ │ Welcome to Claude Code! │ │ Type /help for available commands │ ╰──────────────────────────────────────────╯ > /login Opening browser for authentication... ✓ Logged in as yourname@cpf.co.th
✅ login ครั้งเดียว credentials จะถูกบันทึกไว้ ไม่ต้อง login ใหม่ทุกครั้ง

ขั้นตอนที่ 3 — ตั้งค่า CLAUDE.md

📄 CLAUDE.md คืออะไร?

ไฟล์ instruction พิเศษที่ Claude Code อ่านทุกครั้งที่เริ่ม session ใหม่ ช่วยให้ Claude เข้าใจ context ของ project เช่น โครงสร้างโฟลเดอร์ รูปแบบข้อมูล และกฎการทำงาน ทำครั้งเดียว ใช้ได้ตลอด

1
สร้าง CLAUDE.md สำหรับงาน CPF
ไปที่โฟลเดอร์งาน แล้วรัน /init หรือสร้างไฟล์เอง
Terminal
$ cd /Users/yourname/CPF-Data-Project $ claude > /init ✓ Created CLAUDE.md
ตัวอย่าง CLAUDE.md สำหรับทีม QC CPF
# CPF QC Data Project ## เกี่ยวกับ Project นี้ ระบบวิเคราะห์ข้อมูล QC โรงงาน CPF จาก Excel รายวัน ## โครงสร้างโฟลเดอร์ - /data/raw/ → ไฟล์ Excel จากโรงงาน (ห้ามแก้ไข) - /data/output/ → ผลลัพธ์ที่ประมวลผลแล้ว - /scripts/ → Python scripts - /reports/ → PDF/HTML reports ## รูปแบบข้อมูล - ไฟล์ Excel: ชื่อ QC_YYYYMMDD_PlantCode.xlsx - คอลัมน์หลัก: Date, Plant, Line, Product, Value, Pass/Fail - Encoding: UTF-8-SIG (มี BOM สำหรับภาษาไทย) ## กฎการทำงาน - ใช้ Python 3.10+ และ pandas, openpyxl - ตั้งชื่อฟังก์ชันเป็น snake_case ภาษาอังกฤษ - comment ภาษาไทย เพื่อให้ทีมอ่านเข้าใจ - ทดสอบด้วย sample data ก่อน run จริงเสมอ

A
Lab 1 — สร้าง Python Script วิเคราะห์ข้อมูล QC
ให้ Claude Code เขียน script อ่าน Excel และสรุปผล
Prompt ที่พิมพ์ใน Claude Code
เขียน Python script ชื่อ analyze_qc.py ที่: 1. อ่านไฟล์ Excel ทุกไฟล์ในโฟลเดอร์ data/raw/ 2. รวม column: Date, Plant, Line, Product, Value, Pass/Fail 3. คำนวณ Pass Rate (%) แยกตาม Plant และ Product 4. บันทึกผลลัพธ์เป็น data/output/qc_summary.xlsx 5. แสดง summary ในหน้า terminal พร้อม emoji ✅ ❌ ใช้ pandas และ openpyxl, comment เป็นภาษาไทย

B
Lab 2 — สร้าง Automated Report ส่ง Email
ต่อยอด Lab 1 ให้ส่ง HTML report ทาง email อัตโนมัติ
Prompt ที่พิมพ์ใน Claude Code
ต่อยอดจาก analyze_qc.py โดย: 1. สร้าง HTML report สวยงาม มีตารางและสีแดง/เขียวตาม Pass/Fail 2. บันทึกเป็น reports/qc_report_YYYYMMDD.html 3. ส่ง email อัตโนมัติด้วย smtplib ไปยัง qc-team@cpf.co.th - Subject: "[CPF QC] Daily Report {date}" - แนบไฟล์ Excel ด้วย 4. ใส่ try-except ทุก step และ log error ลงไฟล์ error.log

C
Lab 3 — วิเคราะห์โค้ดที่มีอยู่แล้ว
ให้ Claude Code อ่านและปรับปรุงโค้ดเก่าของทีม
  • 1นำโค้ดหรือ script Python ที่ทีมใช้งานอยู่มาวางในโฟลเดอร์ project
  • 2รัน Claude Code แล้วพิมพ์ prompt ด้านล่าง
  • 3Claude จะอ่านไฟล์ทั้งหมด วิเคราะห์ และให้คำแนะนำ
Prompt วิเคราะห์โค้ดเก่า
อ่านไฟล์ทุกไฟล์ในโฟลเดอร์ scripts/ แล้ว: 1. สรุปว่าแต่ละไฟล์ทำอะไร อธิบายเป็นภาษาไทย 2. หา bug หรือ error ที่อาจเกิดขึ้น 3. แนะนำ 3 วิธีปรับปรุงประสิทธิภาพ 4. บอกว่ามีไฟล์ซ้ำซ้อนหรือไม่จำเป็นหรือเปล่า
Prompt แปลงโค้ดให้ง่ายขึ้น
แก้ไข old_script.py โดย: - ลบโค้ดที่ไม่ได้ใช้งาน - เพิ่ม comment ภาษาไทยทุก function - แบ่งเป็น function ย่อยๆ แทนโค้ดยาวๆ บรรทัดเดียว - เพิ่ม error handling ที่ครอบคลุม - ทดสอบว่ายังให้ผลลัพธ์เดิมหลังแก้ไข

🎯 Prompt ที่ดีสำหรับ Claude Code

  • บอก context — "ไฟล์ Excel ใน data/raw/ มี header: Date, Plant, Value, Pass/Fail"
  • บอก output ที่ต้องการ — "บันทึกเป็น output/summary.xlsx และแสดงผลใน terminal"
  • บอก constraint — "ใช้แค่ pandas ไม่ใช้ library อื่น, Python 3.10+"
  • ขอ test ด้วย — "ทดสอบกับ sample data 5 แถวก่อน แล้วค่อยรันจริง"
  • ขอ error handling — "เพิ่ม try-except ทุก step พร้อม log ไฟล์"
💡 ใช้ Plan Mode ก่อนเสมอกับงานสำคัญ — กด Shift+Tab สองครั้งเพื่อให้ Claude แสดงแผนการทำงานก่อนแก้ไขไฟล์จริง ป้องกันความผิดพลาด
