สร้าง database.js — Schema และ Query Functions
Claude จะออกแบบ table และเขียน functions ทุกอย่าง
💬 พิมพ์ใน Claude Code
สร้าง database.js ที่มี: 1. เชื่อมต่อ SQLite ไฟล์ชื่อ incidents.db 2. สร้าง table incidents ถ้ายังไม่มี: - id INTEGER PRIMARY KEY AUTOINCREMENT - date TEXT NOT NULL (รูปแบบ YYYY-MM-DD) - plant TEXT NOT NULL (เช่น "โรงงาน A", "โรงงาน B") - line TEXT NOT NULL (เช่น "Line 1", "Line 2") - type TEXT NOT NULL (เช่น "คุณภาพสินค้า", "เครื่องจักร", "ความปลอดภัย") - description TEXT NOT NULL - severity TEXT NOT NULL (Low/Medium/High) - status TEXT DEFAULT 'Open' (Open/In Progress/Resolved) - created_at DATETIME DEFAULT CURRENT_TIMESTAMP - updated_at DATETIME DEFAULT CURRENT_TIMESTAMP 3. Export functions: - getAllIncidents(filters) — รองรับ filter plant, status, severity - createIncident(data) - updateStatus(id, status) - getStats() 4. ใส่ sample data 5 รายการ (สำหรับ demo) แทรกแค่ถ้า table ว่างเปล่า comment อธิบายทุก function เป็นภาษาไทย

⚠️ ข้อจำกัด SQLite บน Vercel — สำหรับ Production จริง
Vercel เป็น Serverless — ไฟล์ incidents.db จะถูก reset ทุกครั้งที่ deploy ใหม่ เหมาะสำหรับ demo เท่านั้น
ถ้าต้องการใช้งานจริง ให้ต่อฐานข้อมูลภายนอก:
Vercel Postgres
ฟรี 60 ชั่วโมง/เดือน · เชื่อมใน Vercel Dashboard ได้เลย
Supabase
PostgreSQL ฟรี 500MB · ใช้ REST API หรือ SDK
PlanetScale
MySQL ฟรี · ใช้งานง่ายกับ Node.js
💡 ให้ Claude Code ช่วย migrate: แก้ database.js ให้ใช้ Vercel Postgres แทน SQLite

📁 โครงสร้างไฟล์สุดท้าย

cpf-incident-tracker/ ├── server.js ← Express server + API routes ├── database.js ← SQLite connection + query functions ├── incidents.db ← SQLite database file (auto-created) ├── package.json ← dependencies + scripts ├── CLAUDE.md ← AI instructions สำหรับ project นี้ ├── README.md ← วิธีติดตั้งและรัน ├── .gitignore ← ไม่รวม node_modules, .env ├── .env ← LINE_TOKEN (ถ้าใช้ feature alert) └── public/ ├── index.html ← หน้า Dashboard หลัก ├── report.html ← หน้ากรอก Incident ใหม่ └── app.js ← JavaScript logic ทั้งหมด
