n8n Local — Docker, Port, Volume, Compose

ติดตั้ง n8n Local ด้วย Docker พร้อมอธิบาย Port Mapping, Volume Mapping, docker-compose.yml ทุก field และ 4 Workflows CPF

DockerPort MappingVolumedocker-composePostgreSQL60 นาที
✅ ข้อดี n8n Local
• ฟรีไม่จำกัด execution
• ข้อมูลอยู่ในองค์กร
• ไม่มีค่า subscription

ขั้นตอนที่ 1 — ติดตั้ง Docker Desktop

1

จำเป็นต้องมีก่อนใช้ n8n Local

  • 2เลือก OS: Windows (AMD64) หรือ macOS (Apple/Intel)
  • 3รัน installer ทำตามขั้นตอน → Restart เครื่อง

ขั้นตอนที่ 2 — Port Mapping คืออะไร?

2
อธิบาย -p HOST:CONTAINER
Flag สำคัญในทุก docker command

Port เปรียบเหมือน ประตูบ้าน — Container มีประตูของตัวเอง (port 5678) แต่ต้องบอก Docker ว่าจะเชื่อม "ประตูเครื่องเรา" กับ "ประตู Container" ยังไง

วิธีอ่าน -p flag
docker run -p 5678 : 5678 n8nio/n8n ↑ ↑ │ └─ port ของ Container (n8n ฟัง port นี้เสมอ) └─────────── port ของเครื่องเรา (Host) เข้าใช้งาน → เปิดเบราว์เซอร์ http://localhost:5678
ตัวอย่างเข้าผ่านเมื่อไหร่ใช้
-p 5678:5678localhost:5678ปกติ ✅ แนะนำ
-p 8080:5678localhost:8080ถ้า 5678 ถูกโปรแกรมอื่นใช้
-p 127.0.0.1:5678:5678localhost:5678ปลอดภัยสุด — เครื่องอื่นใน LAN เข้าไม่ได้
-p 0.0.0.0:5678:5678IP_เครื่อง:5678เปิดให้ทีมใน LAN เข้าได้
💡 Production CPF: ใช้ 127.0.0.1:5678:5678 แล้วตั้ง Nginx Reverse Proxy เปิดออกภายนอก

ขั้นตอนที่ 3 — Volume Mapping คืออะไร?

3
อธิบาย -v และ volumes — ป้องกันข้อมูลหาย
ไม่ตั้ง Volume = ข้อมูลหายทุกครั้งที่ลบ Container

Container เปรียบเหมือน กล่องทราย — ข้อมูลภายในหายทันทีเมื่อลบ Container
Volume คือ โฟลเดอร์พิเศษนอก Container ทำให้ Workflow และ Credentials รอดเสมอ

Volume Mapping — 2 แบบ
── แบบที่ 1: Bind Mount — ชี้ไปโฟลเดอร์จริงบนเครื่อง ────── -v /home/cpf/n8n-data : /home/node/.n8n ↑ ↑ โฟลเดอร์บน Host โฟลเดอร์ใน Container (คงที่เสมอ) Windows: -v C:/cpf-data:/home/node/.n8n ── แบบที่ 2: Named Volume — Docker จัดการให้ ────────────── -v n8n_data : /home/node/.n8n ↑ ↑ ชื่อ Volume โฟลเดอร์ใน Container Docker สร้างและเก็บไว้ใน /var/lib/docker/volumes/
ประเภทตัวอย่างข้อดีข้อเสีย
Bind Mount-v /cpf/data:/home/node/.n8nเห็นไฟล์ได้โดยตรงpermission issue บน Windows
Named Volume-v n8n_data:/home/node/.n8nง่าย ไม่มี permission issueดูไฟล์ต้องใช้ docker cp
Read-only-v ./files:/cpf_data:roปลอดภัย Container อ่านได้อย่างเดียวแก้ไขไฟล์จาก container ไม่ได้
📌 /home/node/.n8n คือที่เก็บอะไร?

/home/node/.n8n/database.sqlite — Workflows, Credentials, Settings ทั้งหมด
/home/node/.n8n/config — ไฟล์ configuration
/home/node/.n8n/nodes/ — Custom nodes ที่ติดตั้งเพิ่ม

→ Volume mapping ที่ path นี้เพียงอย่างเดียวก็พอสำหรับ backup ทุกอย่าง
✅ แนะนำ Named Volume สำหรับมือใหม่ เพราะ Docker จัดการให้ทั้งหมด ไม่มี permission error

ขั้นตอนที่ 5 — Docker Compose (แนะนำสำหรับ CPF)

🆚 ทำไม docker compose ดีกว่า docker run?

❌ docker run
• พิมพ์ command ยาวทุกครั้ง
• จำยาก ผิดง่าย
• ไม่มี version control
• share ทีมลำบาก
✅ docker compose
• docker compose up -d คำสั่งเดียว
• config อยู่ในไฟล์ แก้ง่าย
• เก็บใน Git ได้
• รวม services หลายตัวได้
5A
สร้างไฟล์ docker-compose.yml — Basic
1 service: n8n อย่างเดียว
  • 1สร้างโฟลเดอร์: mkdir cpf-n8n && cd cpf-n8n
  • 2สร้างไฟล์ชื่อ docker-compose.yml (ตรงๆ ไม่มีนามสกุลอื่น)
  • 3วาง content จาก prompt ด้านล่างลงไปในไฟล์
  • 4รัน: docker compose up -d
  • 5ตรวจสอบ: docker compose ps
  • 6เปิด http://localhost:5678
docker-compose.yml Basic — อธิบายทุก field
version: "3.8" # Compose format version services: # ประกาศ services ทั้งหมด n8n: # ชื่อ service (ตั้งเองได้) image: n8nio/n8n:latest # Docker image ที่ใช้ restart: always # รีสตาร์ทอัตโนมัติทุกกรณี # ค่าอื่น: no, on-failure, unless-stopped ports: # Port Mapping section - "5678:5678" # "host_port:container_port" environment: # Environment Variables - GENERIC_TIMEZONE=Asia/Bangkok # timezone ไทย - N8N_BASIC_AUTH_ACTIVE=true # เปิดระบบ login - N8N_BASIC_AUTH_USER=admin # username - N8N_BASIC_AUTH_PASSWORD=cpf2569! # password - N8N_LOG_LEVEL=info # debug/info/warn/error - N8N_EXECUTIONS_DATA_PRUNE=true # ลบ log เก่าอัตโนมัติ - N8N_EXECUTIONS_DATA_MAX_AGE=168 # เก็บ log 7 วัน volumes: # Volume Mapping section - n8n_data:/home/node/.n8n # named volume → เก็บ workflows - ./local_files:/files # bind mount → โฟลเดอร์ในเครื่อง volumes: # ประกาศ named volumes n8n_data: # Docker จัดการ volume นี้ให้

5B
docker-compose.yml — Production พร้อม PostgreSQL
เหมาะกับงานจริงที่มี Workflow เยอะ

n8n ใช้ SQLite โดย default — เหมาะทดลอง
Production ที่รัน Workflow หลายตัวพร้อมกัน ควรเปลี่ยนเป็น PostgreSQL

docker-compose.yml Production + PostgreSQL — อธิบายทุก field
version: "3.8" services: # ─── Service 1: PostgreSQL Database ─────────────────────── db: image: postgres:15-alpine # Alpine = ขนาดเล็ก เร็ว restart: always environment: POSTGRES_DB: n8n # ชื่อ database POSTGRES_USER: n8nuser # username POSTGRES_PASSWORD: cpf_db_2569! # password volumes: - postgres_data:/var/lib/postgresql/data # เก็บข้อมูล DB networks: - n8n-net # อยู่ใน network เดียวกับ n8n healthcheck: # ตรวจว่า DB พร้อมก่อน n8n start test: ["CMD-SHELL", "pg_isready -U n8nuser"] interval: 10s # ตรวจทุก 10 วินาที timeout: 5s # timeout 5 วินาที retries: 5 # ลองซ้ำ 5 ครั้ง # ─── Service 2: n8n Application ─────────────────────────── n8n: image: n8nio/n8n:latest restart: always depends_on: # รอ db healthy ก่อน start db: condition: service_healthy ports: - "127.0.0.1:5678:5678" # local-only (ปลอดภัยกว่า) environment: # --- Authentication --- - N8N_BASIC_AUTH_ACTIVE=true - N8N_BASIC_AUTH_USER=admin - N8N_BASIC_AUTH_PASSWORD=cpf2569! # --- Database Connection --- - DB_TYPE=postgresdb # บอก n8n ให้ใช้ PostgreSQL - DB_POSTGRESDB_HOST=db # ชื่อ service (Docker resolve ให้) - DB_POSTGRESDB_PORT=5432 # port PostgreSQL - DB_POSTGRESDB_DATABASE=n8n # ชื่อ database - DB_POSTGRESDB_USER=n8nuser # ต้องตรงกับ POSTGRES_USER - DB_POSTGRESDB_PASSWORD=cpf_db_2569! # ต้องตรงกัน # --- Timezone & Performance --- - GENERIC_TIMEZONE=Asia/Bangkok - N8N_EXECUTIONS_DATA_PRUNE=true - N8N_EXECUTIONS_DATA_MAX_AGE=168 - N8N_CONCURRENCY_PRODUCTION_LIMIT=10 # จำกัด concurrent workflows volumes: - n8n_data:/home/node/.n8n # เก็บ config, credentials - ./cpf_files:/cpf_data:ro # :ro = read-only (ปลอดภัยกว่า) networks: - n8n-net # ─── Named Volumes ───────────────────────────────────────── volumes: postgres_data: # Docker เก็บข้อมูล PostgreSQL ให้ n8n_data: # Docker เก็บ n8n workflows/credentials ให้ # ─── Internal Network ────────────────────────────────────── networks: n8n-net: driver: bridge # container คุยกันภายในได้ แต่ปิดจากภายนอก
📋 docker-compose.yml Production (copy ได้เลย)
version: "3.8"

services:
  db:
    image: postgres:15-alpine
    restart: always
    environment:
      POSTGRES_DB: n8n
      POSTGRES_USER: n8nuser
      POSTGRES_PASSWORD: cpf_db_2569!
    volumes:
      - postgres_data:/var/lib/postgresql/data
    networks:
      - n8n-net
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U n8nuser"]
      interval: 10s
      timeout: 5s
      retries: 5

  n8n:
    image: n8nio/n8n:latest
    restart: always
    depends_on:
      db:
        condition: service_healthy
    ports:
      - "127.0.0.1:5678:5678"
    environment:
      - N8N_BASIC_AUTH_ACTIVE=true
      - N8N_BASIC_AUTH_USER=admin
      - N8N_BASIC_AUTH_PASSWORD=cpf2569!
      - DB_TYPE=postgresdb
      - DB_POSTGRESDB_HOST=db
      - DB_POSTGRESDB_PORT=5432
      - DB_POSTGRESDB_DATABASE=n8n
      - DB_POSTGRESDB_USER=n8nuser
      - DB_POSTGRESDB_PASSWORD=cpf_db_2569!
      - GENERIC_TIMEZONE=Asia/Bangkok
      - N8N_EXECUTIONS_DATA_PRUNE=true
      - N8N_EXECUTIONS_DATA_MAX_AGE=168
      - N8N_CONCURRENCY_PRODUCTION_LIMIT=10
    volumes:
      - n8n_data:/home/node/.n8n
      - ./cpf_files:/cpf_data:ro
    networks:
      - n8n-net

volumes:
  postgres_data:
  n8n_data:

networks:
  n8n-net:
    driver: bridge

ขั้นตอนที่ 6 — คำสั่ง Docker Compose ที่ใช้บ่อย

6
Commands ครบชุด
จำแค่ 5-6 คำสั่งนี้ก็พอ
คำสั่งหน้าที่เมื่อไหร่ใช้
docker compose up -dเริ่ม services ทั้งหมด backgroundครั้งแรก / หลัง restart เครื่อง
docker compose downหยุดและลบ container (volume ยังอยู่)หยุดชั่วคราว
docker compose down -vหยุดและลบ container + ลบ volume ด้วย⚠️ ข้อมูลหาย! reset ใหม่
docker compose stopหยุด container (ไม่ลบ)พักชั่วคราว
docker compose startเริ่ม container ที่หยุดไว้หลัง stop
docker compose restartrestart ทุก serviceหลังแก้ config
docker compose psดู status ทุก serviceตรวจสอบปกติ
docker compose logs -f n8nดู logs real-timedebug ปัญหา
docker compose pullดาวน์โหลด image ใหม่เตรียม update
docker compose up -d --force-recreaterecreate container ด้วย image ใหม่หลัง pull

ขั้นตอนที่ 8 — Troubleshooting ปัญหาบ่อย

8
วิธีแก้ปัญหาที่พบบ่อย
❌ port is already allocated
สาเหตุ: Port 5678 ถูกโปรแกรมอื่นใช้อยู่
แก้ไข: เปลี่ยน - "5679:5678" แล้วเข้า localhost:5679
❌ Permission denied (Windows)
สาเหตุ: Docker Desktop ยังไม่ได้รับ permission
แก้ไข: Settings → Resources → File Sharing → เพิ่ม drive ที่ใช้
❌ Container keeps restarting
สาเหตุ: Environment variable ผิด หรือ DB ยังไม่พร้อม
แก้ไข: docker compose logs n8n ดู error แล้วแก้
❌ Schedule ไม่ทำงานตามเวลา
สาเหตุ: ไม่ได้ตั้ง timezone หรือ Workflow ไม่ได้ Active
แก้ไข: ตรวจสอบ GENERIC_TIMEZONE=Asia/Bangkok และ toggle สีเขียว
❌ Workflows หายหลัง restart
สาเหตุ: ไม่ได้ตั้ง Volume
แก้ไข: เพิ่ม volumes section ใน compose แล้ว recreate
❌ Cannot connect to db
สาเหตุ: Password ไม่ตรงกัน หรือ healthcheck ยังไม่ผ่าน
แก้ไข: ตรวจ POSTGRES_PASSWORD = DB_POSTGRESDB_PASSWORD ต้องตรงกันทุกตัวอักษร

📋 Code Node — คำนวณ Stock Alert
const items = $input.all();
const alerts = [];
for (const item of items) {
  const cur = parseFloat(item.json['สต็อกปัจจุบัน (ตัน)']) || 0;
  const min = parseFloat(item.json['ปริมาณขั้นต่ำ (ตัน)']) || 0;
  if (cur < min) {
    alerts.push({ json: {
      material: item.json['ชื่อวัตถุดิบ'],
      current: cur, minimum: min,
      level: cur < min * 0.5 ? 'CRITICAL' : 'WARNING',
      should_order: Math.round(min * 1.5 - cur)
    }});
  }
}
return alerts.length > 0 ? alerts : [{json:{level:'OK'}}];
